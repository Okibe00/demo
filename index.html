Welcome to git and github

Git is a distributed version control system.

knowledge of git and github is essential for effective collaboration during software develpoment.

Here's what i learnt from todays session

  • git restore file : removes changes from the working area.
  • git rm --cached :removes all files in staging area
  • git rm --cached file :removes specified file
  • restore --staged file :removes staged file
  • git commit --amend :undo's a commit by replacing it with the currently staged file.
  • git tag :Adds a tag to commit
  • git remote :displays remote info.
  • git remote show :displays a verbose remote info.
  • git remote fetch remoteName :gets current info in remote repo.
  • git remote add shortname url :adds a remote repo with custom name.
  • git push remotename :pushes data to remote repo.
  • git pull :gets data from remote repo, and merges automatically.
  • git branch branchname :creates a new branch
  • git checkout branch :switch to branch
  • git branch :view available branch
  • git log branch :log commits on branch regardless of current branch.
  • git log --all :log info on every branch available.
  • git switch brnach name :switch branch
  • git switch -c branch : create and switch branch

Working with remotes

Remotes are repositories that aren't on your local machine, for most part there maybe on hosting services like github.

When the command git clone is run a remote name called origin is given to the remote server on your local machine origin is just a default name and can be changed to what ever with git clone -o name .

Displaying available remotes

The command git remote show origin will display all available remote repo.
